# Holiday & Event Calendar (R14)

> **零 API**。静态数据,内嵌在 SKILL references。
> 100+ 营销节点 + 节日 + 行业事件。SKILL 在节点前 7 天主动建议蹭热点。

## 设计

- 节点分 3 类:**法定节假日** / **电商节** / **行业事件**
- 每个节点带:`date` `category` `platforms_best_fit` `content_themes` `days_advance`
- `/daily` 跑时,SKILL 检查当前日期 → 未来 7 天内节点 → 在内容卡候选里加节点关联

## 数据格式

每条节点用 YAML 存,SKILL 读 `.holiday-calendar.yaml`:

```yaml
- date: 02-14
  name: 情人节
  category: holiday
  platforms: [小红书, 抖音, Instagram]
  themes: [礼物, 情侣, 单身, 约会]
  advance: 7

- date: 11-11

```

## `/daily` 集成逻辑

```python
# SKILL 跑 /daily 时
def check_holidays(today):
    upcoming = []
    for d in range(0, 8):  # 未来 7 天
        day = today + timedelta(days=d)
        matches = find_holiday(day)
        for h in matches:
            if d <= h.advance:  # 提前期足够
                upcoming.append((day, h))
    return upcoming

# 输出
if upcoming:
    print("📅 即将到来的节点:")
    for day, h in upcoming:
        print(f"  {day} ({d}天后): {h.name} - 主题:{h.themes}")
        print(f"     建议:做 1 期 '{h.themes[0]}' 相关内容")
```

## 节点内容模板(每个节点)

```markdown
## {name} 内容建议

### 时机
- 提前 {advance} 天发预热
- 当天发主战场
- 后 1-2 天发延续(回放/总结)

### 适合主题
- {theme 1}
- {theme 2}
- {theme 3}

### 历史参考
(用户在历年 {name} 发了什么,效果如何)

### 蹭点建议
- 用 LLM 找"你 + {name}"的独特角度
- 不要"祝大家 {name} 快乐"这种烂大街
- 要"作为 XX,我在 {name} 想..."
```

## 注意事项

1. **不要节日当天发**:大节点(如 618/双 11)当天流量被广告淹没,前 1-2 天效果更好。
2. **避免政治敏感**:党的生日、建军节等,只发正能量,不评论。
3. **跨文化注意**:做海外平台时,部分国内节日不适用(如春节),部分西方节日要本地化。
4. **个人 vs 平台**:不是所有节日都对你的 niche 相关,SKILL 应该让 LLM 评估相关性,只推送相关的。

## Common Pitfalls

1. **节点过多**:每天被节点轰炸,失去焦点。SKILL 应让用户选"只关注 3 类节点"。
2. **强行蹭**:不相关的节日硬蹭,内容失真。每个节点要让 LLM 判断"是否真的适合我这个 niche"。
3. **节日当天发**:错过最佳提前期,流量被广告盖。SKILL 默认 `advance: 7` 提前准备。
4. **一年只用一次**:每个节点应该做"前/中/后"3 期内容,流量分摊。
