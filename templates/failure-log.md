# R12 失败案例库

> **零 API**。SKILL 内部 LLM 帮用户总结教训 + 主动避免。
> 翻车内容记下来,SKILL 在 `/daily` 时主动避开同类错误。

## 触发

```
/failure add <card_id> --reason="标题党被限流"
/failure list --platform=小红书
/failure patterns              # 让 SKILL 总结常见错误
```

## 数据模型

```yaml
---
type: failure
id: FAIL-2026-06-17-001
date: 2026-06-17
card_id: "[[2026-06-15-002]]"
platform: 小红书
topic: 5 个被高估的 AI 工具
failure_reason: 标题党(用"震惊体")
failure_category: [标题党, 算法惩罚]
actual_views: 80            # 远低于预测
predicted_views: 5000-15000
engagement_rate: 0.5%
diagnosis: "标题用'震惊'词,被平台识别为标题党,流量被限"
lesson: "避免'震惊'/'必看'/'X 个'等夸张词"
severity: high               # low / medium / high
status: analyzed
tags: [标题党, 限流, AI工具]
---
```

## 失败分类(自动)

SKILL 收到 `/failure add` 后让 LLM 分类:

| 分类 | 典型表现 |
|---|---|
| **标题党** | 平台限流, 播放 < 预测 30% |
| **时段错** | 完播率正常但播放低,发布在低谷 |
| **钩子弱** | 高跳出/低完播,前 3 秒流失 |
| **内容空洞** | 互动低(评论/点赞都少) |
| **同质化** | 平台检测重复,限流 |
| **争议性** | 评论多但都是负面的,伤账号权重 |
| **违规** | 直接违规,下架/警告 |
| **设备/技术** | 画面糊、声音小、字幕错位 |

## SKILL 主动避免机制

`/daily` 跑时,SKILL:
1. 读 `_全局/07-失败案例.md` 找 `severity: high` 的案例
2. 拉所有 `failure_category`
3. 检查今日生成的卡是否触发同类问题
4. 警告: "⚠️ 你今天生成的卡 #3 用了'5 个'夸张数字,上月 [FAIL-001] 因同类标题被限流。建议改。"

## 失败模式学习

`/failure patterns` 跑时,SKILL 用 LLM 总结:

```
📊 你的失败模式分析 (本月 7 次失败)

1. **标题党 (3 次, 43%)**
   - 触发词: "震惊" "必看" "99% 的人" "X 个" "不看后悔"
   - 后果: 平均播放下降 75%
   - 建议: 标题用具体数字,不要用夸张词

2. **凌晨发布 (2 次, 29%)**
   - 触发时段: 0-6 点
   - 后果: 完播率 15% vs 正常 45%
   - 建议: 用定时发布功能,绝不手发凌晨

3. **钩子铺垫 (2 次, 29%)**
   - 共同点: 开头 5 秒是"大家好我是 XX"自我介绍
   - 后果: 3 秒跳出率 65%
   - 建议: 钩子第一句直接抛痛点/反差
```

写入 `_全局/00-总档案.md` 的"已知陷阱"段,影响未来所有自动生成。

## 失败案例库 vs SKILL 自检

- **失败案例库** (R12): 单条翻车的内容
- **SKILL 自检** (R10): 整月趋势

两者结合:月报里"Bottom 3 内容"段从失败案例库拉数据。

## 状态机

```yaml
status: new                # 用户刚加,未分析
status: analyzed           # SKILL 已分析原因
status: resolved           # 已应用规避
status: recurring          # 反复出现,需特别关注
```

## Common Pitfalls

1. **失败不记录**:用户看播放低就过,下个月又犯同样错。`/failure add` 应该在用户沮丧时(< 30 秒)。
2. **不归因**:只记"播放低"不记原因 → 无法系统化改进。SKILL 必须让 LLM 给"原因"和"教训"。
3. **过度自责**:不是所有失败都是内容问题,有时是算法/时段/运气。SKILL 给出"可控 vs 不可控"判断。
4. **不复盘**:失败后 3 个月还没看 → 写完不读。SKILL 每月 `/monthly` 强制纳入。
