# 平台特定模板 (Platform-Specific Templates)

> Per-platform 00-06 templates for the 9 supported platforms.
> Files in this directory are the SKILL-side templates; they get copied to the user's vault at `博主计划/{region}/{platform}/` during `/onboard`.

## Why this exists

The SKILL's main `templates/` directory has generic templates (`daily-content-card.md`, `user-profile.md`, etc.) that are platform-agnostic. But each platform has its own:

- Algorithm weights
- Optimal length / format
- Best publish times
- Monetization paths
- Engagement rate benchmarks

Without platform-specific 00-06 files, users would have to research each platform from scratch. This directory solves that.

## Layout

Each platform has 4 files (matching the `00-06` structure):

| File | Purpose |
|---|---|
| `00-profile.md` | Platform basics, account, visual style, posting rhythm, tags, monetization |
| `01-direction.md` | Platform-specific niche, content matrix, hook formulas, viral references |
| `02-roadmap.md` | 6-month M1-M6 plan with platform-specific weekly milestones |
| `06-dashboard.md` | Per-platform metrics table + Dataview query |

`03-trends/`, `04-cards/`, `05-weekly/` directories are platform-specific but don't need templates (the content is generated by `/daily` and `/review`).

## Available platforms

### 国内 (Chinese)
- `douyin/` — 抖音 (15-60s vertical, 完播率-driven)
- `bilibili/` — B站 (5-20min long video, 三连率-driven)
- `wechat-channels/` — 视频号 (1-3min, social-distributed)
- `wechat-mp/` — 公众号 (1500-3000 word articles, 打开率-driven)

### 国外 (International)
- `tiktok/` — TikTok (15-90s vertical, FYP algorithm)
- `instagram/` — Instagram (Reels + carousels + Stories, save-rate-driven)

### Built-in (no separate directory, in main templates/)
- 小红书 (RedNote) — `templates/platform-profile.md` + niche/framework references
- YouTube — same
- X (Twitter) — same

## Usage

When user runs `/onboard` and selects platforms:

```bash
# Generic flow (no platform-specific template)
SKILL copies templates/user-profile.md → vault/_global/00-profile.md
SKILL copies templates/daily-content-card.md → vault/.../04-cards/

# Platform-specific flow
SKILL reads templates/platforms/{platform}/00-profile.md
SKILL substitutes {{USER_NAME}}, {{NICHE}}, {{TARGET_AUDIENCE}} etc.
SKILL writes result → vault/{region}/{platform}/00-平台档案.md
```

Variables to substitute (frontmatter `template_for: ...` declares the schema):

```
{{USER_NAME}}      — display name
{{NICHE}}          — chosen content vertical
{{SUB_NICHE}}      — micro-niche within the vertical
{{TARGET_AUDIENCE}}— primary audience (age, gender, geo)
{{MONETIZATION}}   — primary revenue path
{{LANGUAGE}}       — content language (中文 / English / etc.)
```

## Adding a new platform

1. Create `templates/platforms/{name}/` directory
2. Write 4 files matching the platform's specifics
3. Add entry to `references/platform-specs.md`
4. Update `SKILL.md` Platform Coverage table
5. Run `python scripts/skill-lint.py --strict`

See existing platforms as templates — they follow the same structure.
