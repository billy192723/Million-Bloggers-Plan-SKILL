# R13 评论话术库

> **零 API**。SKILL 内部 LLM 适配场景。
> 评论区常用回复模板,按场景分类,可直接复制。

## 触发

```
/reply <comment> --tone=友好    # 给一个回复建议
/reply <comment> --tone=专业
/reply <comment> --tone=搞笑
```

## 场景分类(共 6 大类)

### 1. 感谢/认可型

> 用户的评论:"太有用了!""学到了!""爱了爱了"

回复模板(3 选 1):

```
A. 真诚感谢 + 引导关注
   "谢谢你!能帮到你很开心 😊 关注我,后面还有更多 XX 类的实操分享~"

B. 邀请互动
   "感谢支持!你之前用过 XX 吗?效果怎么样?评论区聊聊 👇"

C. 预告下期
   "很高兴帮到你!下期我会讲 [相关主题],想看吗?评论区扣 1 提前解锁 🔓"
```

### 2. 提问型

> 用户的评论:"这个工具在哪下载?""多少钱?"

回复模板:

```
A. 直接给答案
   "[工具名] 官网是 [URL],有免费版可以先试。"

B. 引导深度互动
   "好问题!简单说: [答案]。如果想看详细对比,我可以专门做一期。要吗?评论区扣 1"

C. 转化成私域
   "这个我整理了详细清单,主页有我的联系方式,私我发你 ✉️"
```

### 3. 质疑/反驳型

> 用户的评论:"这个工具不好用吧""我试过不行""博主是不是收了钱"

回复模板(小心处理,负面评论权重高):

```
A. 理性回应
   "[承认不同场景] 你说的对,在 XX 场景下确实一般。我这次主要测的是 [具体场景],可能不全面。回头我专门测一下你这种场景,期待吗?"

B. 求反馈(转质疑为内容)
   "能详细说说你的体验吗?我做内容就是为了让信息更全,你的反馈能帮到其他人。"

C. 不直接回应(适用于恶意评论)
   "感谢分享观点 🤝" + 不再回复
   (让算法看到正面互动,绕开冲突)
```

### 4. 推销/广告型

> 用户的评论:"找我买 XX""加我私聊"

回复模板:

```
A. 礼貌拒绝
   "本号不接受广告合作,有需要可以看我简介里的合作方式 ✉️"

B. 转给商务
   "感谢联系!商务合作请发邮件到 [邮箱],会有专人回复。"

C. 沉默(适用于明显垃圾)
   不回复,直接屏蔽
```

### 5. 故事征集型(钩子)

> 你主动发的评论:"你试过 XX 吗?评论区聊聊"

预期回复 + 你的回复:

```
A. 故事丰富
   "我之前用 [工具] 做了 [结果],但 [踩坑],后来发现 [你的方法]..."
   你的回复: "哇你这个故事太真实了!能展开说说吗?我可以做一期深挖你这种情况 ✨"

B. 简单体验
   "我用过,感觉一般"
   你的回复: "感谢分享!可以具体说说什么场景下觉得一般吗?帮我也避个坑"

C. 反问
   "没试过,但你这么说有点想试"
   你的回复: "那快去试!试了来评论区告诉我效果,我也想知道"
```

### 6. 互动/翻牌型

> 你的内容有"评论区扣 1"或"评论告诉我你的情况"

预期回复 + 你的回复:

```
A. 大量扣 1
   你的回复: "前 10 个扣 1 的我挨个回复!没扣的也来啊~"

B. 详细回答
   你的回复: "[针对每个回答给定制回复,这是高价值互动,算法权重高]"

C. 吐槽/差评
   你的回复: "哈哈哈这也太真实了 😂 你这情况我之前也遇到过,下次专门讲!"
```

## 高级策略

### 触发"二次评论"的话术(算法加权)

> 评论区互动率是 2024-2026 平台主要算法权重之一。
> 关键:让评论者**回你的评论**,而不是单向回复。

```
钩子:
"你最常用的是哪个?评论区告诉我,下期我做对比 🔥"
```

```
你追评:
"看到大家都在用 [X],果然是国民工具!还有什么隐藏好用的小众工具?求推荐!"
```

> 这能让评论者**再次**评论,把 1 个互动变成 2-3 个。

### 争议评论的"高情商回复"

> 不是所有争议都要避免。**正面争议**能拉互动率。

```
争议评论: "这个 AI 工具就是割韭菜"

差回复: "你不懂别乱说"
好回复: "你这种观点我理解。其实工具是不是割韭菜,看用在什么场景。
       我这次测的是 [具体场景],确实有效果。
       你觉得哪些场景下没用?我下次专门测一下。"
```

> 把"反驳"变成"内容机会",算法权重 + 观众好感双收。

### 私域引流(踩线)

> 平台不喜欢直接引流到微信/私信。技巧:
> - 不说"加我微信",说"主页有联系方式"
> - 不在评论区说"私信我",用自动回复关键字触发
> - 把引流入口放在"个人简介",评论里只说"看简介"

```
"想看完整对比的,看简介 ✨"
```

## 回复决策表(SKILL 用)

| 评论类型 | 数量 | 回复优先级 | 模板 |
|---|---|---|---|
| 提问型 | 高 | 必回,优先 | 1.2 |
| 感谢型 | 中 | 必回,简短 | 1.1 |
| 质疑型 | 低 | 必须回,谨慎 | 3.1 或 3.3 |
| 推销型 | 中 | 不回或拒 | 4.1 |
| 故事征集型 | 高 | 必回,定制 | 5.1-5.3 |
| 互动/翻牌型 | 高 | 必回,简短 | 6.1-6.3 |

## 自动回复规则(高级)

```yaml
# 在 _全局/04-推送配置.md 加段:
auto_reply:
  enabled: false                # 默认手动,熟悉后再开
  trigger_keywords: ["教程", "链接", "怎么"]
  reply_template: "感谢留言!你需要的 [X] 主页有详细教程,关注 + 简介可以看到 ✨"
  max_per_day: 20               # 防止被识别为机器人
  only_if_follower: true        # 只回粉丝,减少垃圾
```

## Common Pitfalls

1. **每条都回**:消耗时间且对涨粉帮助有限。**优先回"提问型"和"故事型"**,其他批量处理。
2. **回复太长**:评论区是公共场所,长回复挤占别人空间。**理想 1-2 行,最多 4 行**。
3. **态度太硬**:反驳评论时不能怼,会把粉丝推到对面。
4. **不引导二次互动**:回复完就走 = 浪费算法权重。每条回复结尾引导对方"再评论一次"。
5. **复制粘贴同一句**:评论者能识别,觉得是机器人。**用 LLM 改写,保持人味**。
