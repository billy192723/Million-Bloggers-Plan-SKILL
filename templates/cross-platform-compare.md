# R9 双平台 A/B 对比

> **零 API**。SKILL 逻辑 + 用户回填数据。
> 同 idea 发到 2 平台 → 1 周后对比 → 给迁移建议。

## 触发

```
/xplatform <idea> --platforms=小红书,B站
/xplatform <card_id>          # 已有卡,加第 2 平台
```

## 工作流

1. SKILL 标记同 idea 的多张卡 `cross_platform_group: "{group_id}"`
2. 用户发布各平台
3. 7d 后用户回填 actual 数据
4. SKILL 算 ROI,对每平台评分

## 对比维度

| 维度 | 计算 |
|---|---|
| 绝对流量 | 7d 累计播放 |
| 互动质量 | 互动率 / 完播率 |
| 涨粉效率 | 新增粉丝 / 总播放 |
| 制作成本 | 制作时长 × 难度系数 |
| 内容寿命 | 7d 后是否还在涨(长尾) |
| 综合 ROI | 互动质量 × 涨粉效率 / 制作成本 |

- **首选平台**: 小红书 (高 ROI,涨粉快)

## 跨平台对比决策规则(SKILL 用)

| 信号 | 建议动作 |
|---|---|
| 平台 A ROI 持续 A 级 ≥ 4 周 | 加投 A,降投其他 |
| 平台 B ROI 长期 C 级 ≥ 8 周 | 暂停 B,资源切走 |
| 新平台前 4 周 ROI 模糊 | 给 3 个月观察期,不平仓 |
| 平台 C 长尾持续涨 | 保留 C(未来可能成为主战场) |
| 2 平台数据接近 | 看制作成本,选成本低的 |

## 数据收集

7 天后用户输入(可手填,SKILL 解析):

```
小红书 播放 8500 互动率 5.2 涨粉 89
B站 播放 1200 互动率 8.5 涨粉 22 完播 45
YouTube 播放 320 互动率 6.1 涨粉 5
```

SKILL 自动算 ROI 并填表。

## Common Pitfalls

1. **时间窗口不一致**:某平台发完后 24h 内另一平台没发,数据不可比。SKILL 应该要求同 idea 在 48h 内发完。
2. **算法冷启动差异**:新平台新账号前 3 条数据噪音大,要在对比里标注。
3. **内容适配不均**:同 idea 没做平台原生改写,直接复用 → 数据不公平。SKILL 应先用 R3 改写。
4. **粉丝基线不同**:平台 A 1k 粉 / 平台 B 100 粉,绝对流量差异不能只看数字,要算"每千粉的播放数"。
